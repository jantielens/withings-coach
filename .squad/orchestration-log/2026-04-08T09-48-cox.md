### Key Decisions

1. **Context-in-Prompt Strategy** — Rejected RAG. BP data volume (~15K tokens/year) fits comfortably in 128K context window. No vector DB, embedding pipeline, or retrieval complexity needed.

2. **System Architecture** — VS Code-style split layout (health dashboard left, chat panel right). Single `POST /api/chat` endpoint. No server-side chat history. Client owns message state.

3. **Azure Resources** — 3 total: AI Foundry hub + project + GPT-4o deployment. No search, vector DB, or Redis.

4. **Authentication** — Managed identity via `DefaultAzureCredential`. Local dev uses `az login` credentials. Production uses system-assigned identity on App Service.

5. **Streaming** — Server-Sent Events (SSE) from `/api/chat`. Vercel AI SDK handles conversion from Azure OpenAI streaming to SSE.

6. **Layout** — Collapsible chat panel; default collapsed on mobile, expanded on desktop. Resizable via drag handle.

7. **System Prompt** — Refactor existing `buildBPPrompt()` into `buildChatSystemMessage()`. Add conversational instructions, date awareness, scope boundaries. Rebuild on every request to include latest data.

8. **Risk Assessment** — Biggest blocker: managed identity auth spike in Week 1. If `DefaultAzureCredential` → Azure OpenAI streaming fails in Next.js server, fallback to API key.
