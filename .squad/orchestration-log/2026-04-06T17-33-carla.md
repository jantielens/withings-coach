## Issues Flagged

### 🚨 Data Quality: BP Input Validation

**From:** Carla discovery during testing  
**Issue:** `classifyBloodPressure(0, 0)` returns "normal"; negative values don't throw

**Details:**
- Zero and negative values fall below all thresholds, thus classify as "normal"
- Mathematically correct but clinically meaningless
- Risk: sensor glitch sending `0/0` displays as "Normal"
- Phase 2 impact: LLM coaching agent could interpret zero as healthy; doctor view showing `0/0 — Normal` erodes trust

**Recommendation:** Add input validation guard in `classifyBloodPressure()` before Phase 2. Physiologically reasonable bounds: systolic ≥ 40 mmHg, diastolic ≥ 20 mmHg.

**Status:** Deferred to Phase 2; MVE safe (Withings API won't send zeroes)
